Báo cáo quy trình kỹ thuật & Luồng xử lý FlashStudy (Full Version)

1. Học tập Flashcard Tương tác

Giúp người dùng ghi nhớ kiến thức thông qua bộ thẻ lật 3D, hỗ trợ điều hướng và theo dõi tiến độ học tập trực quan.

Bước 1 Người dùng

Nhấn vào thẻ Flashcard trên màn hình hoặc sử dụng nút "Tiếp theo/Quay lại".

Bước 2 Client (Trình duyệt)

Gửi yêu cầu GET tới file flashcard.html và thực thi script điều hướng.

Bước 3 Server (Logic)

Server xử lý yêu cầu định tuyến, cung cấp file HTML và dữ liệu thẻ tương ứng.

Bước 4 Database (MongoDB)

Truy vấn vào collection Contents để lấy nội dung các thẻ theo bộ ID.

Bước 5 Server trả lại

Trả về chuỗi JSON hoặc HTML chứa nội dung các mặt thẻ (Term & Definition).

Bước 6 Client hiển thị

Kích hoạt hiệu ứng CSS 3D Flip và cập nhật thanh tiến độ (Thẻ X/Y).

2. Hệ thống Bình luận & Kiểm duyệt

Người dùng có thể đánh giá bộ thẻ bằng sao và để lại nhận xét. Hệ thống tích hợp lọc từ ngữ không phù hợp.

Bước 1 Người dùng

Nhập nội dung vào ô bình luận, chọn số sao và nhấn "Gửi".

Bước 2 Client (Trình duyệt)

Script kiểm tra Blacklist từ xấu. Gửi POST tới /api/comments.

Bước 3 Server (Logic)

CommentController nhận yêu cầu, kiểm tra Session và Sanitize nội dung.

Bước 4 Database (MongoDB)

Lệnh comment.save() lưu thông tin vào collection Comments.

Bước 5 Server trả lại

Phản hồi JSON { success: true } kèm đối tượng bình luận vừa tạo.

Bước 6 Client hiển thị

JS thêm bình luận vào UI tức thì mà không cần tải lại trang.

3. Quản lý Thư viện cá nhân (LocalStorage & API)

Cho phép người dùng tạo các bộ sưu tập thẻ riêng, lưu trữ tiến độ học tập và quản lý danh mục bài học.

Bước 1 Người dùng

Nhấn "Tạo bộ thẻ mới" trong trang Library và nhập tên thư mục.

Bước 2 Client (Trình duyệt)

Khởi tạo đối tượng Collection và lưu vào LocalStorage để truy xuất nhanh.

Bước 3 Server (Logic)

Đồng bộ hóa dữ liệu thư viện từ Client lên Server để lưu trữ vĩnh viễn trên tài khoản.

Bước 4 Database (MongoDB)

Cập nhật danh sách bộ sưu tập trong collection Users hoặc Contents.

Bước 5 Server trả lại

Thông báo trạng thái đồng bộ thành công về cho phía người dùng.

Bước 6 Client hiển thị

Hiển thị thẻ thư mục mới trên Collection Grid với số lượng thẻ là 0.

4. Chỉnh sửa nội dung Flashcard (Admin)

Quản trị viên có thể thay đổi tiêu đề, nội dung của các bộ thẻ đã có để cập nhật kiến thức mới nhất.

Bước 1 Người dùng (Admin)

Nhấn nút "Sửa" tại danh sách quản lý nội dung trong Dashboard.

Bước 2 Client (Trình duyệt)

Gửi yêu cầu GET tới /admin/edit/:id để lấy form chỉnh sửa.

Bước 3 Server (Logic)

AdminController lấy ID từ URL và tìm kiếm nội dung tương ứng trong CSDL.

Bước 4 Database (MongoDB)

Sử dụng findById(id) để lấy dữ liệu thô của flashcard đó.

Bước 5 Server trả lại

Server trả lời bằng cách render('admin/edit') kèm dữ liệu cũ vào Form.

Bước 6 Client hiển thị

Giao diện hiển thị Form chứa dữ liệu cũ, sẵn sàng cho việc nhập liệu mới.

5. Cập nhật dữ liệu hệ thống (API Put)

Lưu các thay đổi sau khi Admin đã chỉnh sửa nội dung bài học vào cơ sở dữ liệu.

Bước 1 Người dùng (Admin)

Nhập dữ liệu mới vào Form và nhấn "Cập nhật nội dung".

Bước 2 Client (Trình duyệt)

Gửi yêu cầu PUT tới /admin/edit/:id kèm theo Body JSON chứa tiêu đề/nội dung mới.

Bước 3 Server (Logic)

Server xác thực quyền quản trị và thực hiện logic cập nhật bản ghi.

Bước 4 Database (MongoDB)

Thực hiện hàm findByIdAndUpdate(id, req.body) để ghi đè dữ liệu cũ.

Bước 5 Server trả lại

Trả về JSON thông báo thành công: { success: true }.

Bước 6 Client hiển thị

Thông báo "Cập nhật thành công!" và tự động chuyển hướng về trang danh sách chính.

6. Gửi phản hồi liên hệ (Contact Form)

Người dùng có thể gửi thắc mắc hoặc góp ý trực tiếp tới đội ngũ phát triển qua hệ thống Mail/Form.

Bước 1 Người dùng

Điền Họ tên, Email và Nội dung tin nhắn vào trang contact.html.

Bước 2 Client (Trình duyệt)

Gửi POST request tới API xử lý tin nhắn của hệ thống.

Bước 3 Server (Logic)

Server nhận tin nhắn, kiểm tra tính hợp lệ của định dạng Email người gửi.

Bước 4 Database (MongoDB)

Lưu thông tin liên hệ vào collection Feedbacks để Admin theo dõi.

Bước 5 Server trả lại

Trả lời HTML thông báo tin nhắn đã được gửi đi thành công.

Bước 6 Client hiển thị

Hiển thị Modal thông báo hoặc chuyển hướng người dùng về trang chủ.
